# WORK_DONE

## Target

Make this Rust project generate audible human speech from Qwen3-TTS text input through the compiled CLI.

Current target command shape:

```powershell
cargo run -p qwen-vox-cli --features cuda -- generate --device cuda --language chinese --speaker vivian --text "你好，這是一段語音測試。" --output out\speech.wav
```

## Current Status

Not complete yet.

The old full-pipeline symptom was full-scale high-frequency noise. Decoder-side clipping has been fixed, but the Rust CLI output still has not been proven to contain human voice. The remaining problem is likely in talker / codec-token generation, not only in the waveform decoder.

Important clarification: `out/python_decoder_test_output.wav` is also not human speech. It is only a PyTorch reference decode of fixed `test_tokens`; those tokens are not known-good speech. Its purpose is numerical decoder comparison, not audible TTS validation.

The first verified upstream human-voice reference is:

- `out/official_qwen3_reference.wav`
- Generated by the official Qwen3-TTS Python model flow from local `weights/hf_original`.
- Text: `你好，這是官方 Qwen3 TTS 參考語音。`
- Speaker/language: `vivian` / `chinese`
- Frames: 50 codec frames, 16 codebooks
- Duration: about 3.976875s
- Audio is confirmed by listening as normal human speech.

Latest validation:

- `python tools\generate_official_reference.py --help` succeeds.
- `cargo check -p qwen-vox-cli --features cuda` succeeds without warnings.

## Completed In This Stage

1. Used codebase-memory MCP to locate the codec decoder, alignment tests, CLI generation path, and current decoder/talker entry points.
2. Compared Rust decoder-only output against an upstream PyTorch tokenizer decoder loaded from the same tokenizer weights.
3. Fixed major tokenizer decoder mismatches:
   - SnakeBeta formula matches upstream log-space parameters: `x + sin^2(exp(alpha) * x) / (exp(beta) + 1e-9)`.
   - Codebook lookup reconstructs EMA codebooks from `embedding_sum / clamp(cluster_usage, 1e-5)`.
   - ConvTranspose cropping and ConvNeXt block order are aligned with upstream behavior.
   - Decoder pre-transformer now uses a causal mask.
4. Verified decoder-only output is no longer full-scale clipped:
   - `out/python_decoder_test_output.wav`: 14805 samples, RMS about -18.28 dB, 0 clipped samples.
   - Rust decoder reference output after fixes: RMS about -18.48 dB, 0 clipped samples in the latest aligned run.
5. Ran a real CLI CUDA generation pass:
   - Output: `out/qwen3_decoder_aligned_16frames.wav`
   - Duration about 1.256875s
   - RMS about -17.486 dB
   - 0 clipped samples
   - This confirms the decoder no longer creates the old full-scale clipping, but does not confirm human voice.
